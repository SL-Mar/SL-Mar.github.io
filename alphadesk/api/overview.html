API Overview

RESTful API for financial data and AI-powered analysis

Base URL

Development: http://localhost:8004/api

API Documentation: http://localhost:8004/docs (Swagger UI)

Architecture

AlphaDesk provides a clean RESTful API built with FastAPI, offering:

  • JSON responses for all endpoints
  • Automatic API documentation via OpenAPI/Swagger
  • CORS enabled for frontend integration
  • Error handling with meaningful status codes
  • Type validation using Pydantic models

API Modules

1. Stocks API

Manage stock projects and retrieve market data from EODHD API.

Method Endpoint Description
POST /api/stocks Create stock project
GET /api/stocks List all stocks
GET /api/stocks/{ticker} Get stock data
GET /api/stocks/{ticker}/eod Get EOD price data
DELETE /api/stocks/{ticker} Delete stock project

View Stocks API Documentation →

2. Signals API

Generate and retrieve AI-powered trading signals using MarketSense AI framework.

Method Endpoint Description
POST /api/signals/generate/{ticker} Generate trading signal
GET /api/signals/history/{ticker} Get signal history
GET /api/signals/latest Get latest signals

View Signals API Documentation →

3. Settings API

Manage application settings and API keys.

Method Endpoint Description
GET /api/settings Get all settings
PUT /api/settings Update settings

View Settings API Documentation →

Authentication

Currently, AlphaDesk is designed for local development and does not require authentication. For production deployments:

  • Implement API key authentication
  • Use JWT tokens for user sessions
  • Enable HTTPS for secure communication
Security Note

The current version is intended for local development only. Do not expose the API to the public internet without implementing proper authentication and security measures.

Response Format

Success Response

{
  "data": {...},
  "status": "success"
}

Error Response

{
  "detail": "Error message",
  "status_code": 400
}

HTTP Status Codes

Code Meaning Description
200 OK Request succeeded
201 Created Resource created successfully
400 Bad Request Invalid request parameters
404 Not Found Resource not found
409 Conflict Resource already exists
500 Internal Server Error Server-side error
503 Service Unavailable External service (EODHD, LLM) unavailable

Rate Limiting

Current rate limits:

  • EODHD API: 100 requests/day (free tier)
  • Signal Generation: 5 requests/minute (recommended due to LLM API costs)
  • Other endpoints: No hard limits for local development

CORS Configuration

CORS is enabled for the following origins:

  • http://localhost:3004 (default frontend)
  • http://localhost:3001 (alternative port)

Interactive API Documentation

AlphaDesk provides automatic interactive API documentation:

  • Swagger UI: http://localhost:8004/docs
  • ReDoc: http://localhost:8004/redoc

Use these interfaces to explore endpoints, test requests, and view response schemas interactively.
