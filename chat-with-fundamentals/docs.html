Overview

Chat with Fundamentals is a full-stack financial research and portfolio management platform that combines AI-powered fundamental analysis with advanced quantitative portfolio optimization.

Built with CrewAI, LangChain, FastAPI, Next.js, PostgreSQL, Redis, and EODHD APIs for comprehensive market data.

üìä Application Overview Screenshot

Note: This is a research engine, not a SaaS platform. It is designed to be local-first and tailored to user needs through continuous refinement of AI and quantitative workflows.

Project Philosophy

Chat with Fundamentals is designed as a research engine to explore how autonomous agents and AI can automate and enhance financial analysis and portfolio management.

Core Principles

  • Modularity: Workflows, agents, and portfolio tools are designed independently for maximum flexibility
  • Reproducibility: All steps (query, fetch, summarize, optimize) are transparent, logged, and testable
  • Minimal Overhead: Lightweight backend with PostgreSQL + Redis for high performance
  • User Ownership: Users bring their own API keys (OpenAI, EODHD) - no vendor lock-in
  • Shares-Based Management: Real-world portfolio tracking with integer share quantities
  • Local-First: All LLM interactions and data are logged locally for reproducibility

Installation & Setup

Prerequisites

  • Python 3.10+ - Backend runtime
  • Node.js 18+ - Frontend development
  • Docker & Docker Compose - For PostgreSQL and Redis
  • EODHD API Key - Free tier available at eodhd.com
  • OpenAI API Key - For AI-powered features

Deployment

Linux/macOS

# 1. Deploy dependencies
bash deploy.sh

# 2. Start database services
docker-compose up -d

# 3. Configure environment
cp .env.model .env
# Edit .env with your API keys

Windows

REM 1. Deploy dependencies
deploy.bat

REM 2. Start database services
docker-compose up -d

REM 3. Configure environment
copy .env.model .env
REM Edit .env with your API keys

Configuration

Edit the .env file with your credentials:

# API Keys
OPENAI_API_KEY=your-openai-key-here
EODHD_API_KEY=your-eodhd-key-here

# LLM Configuration
MODEL_NAME=gpt-4o

# Database
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/chatfundamentals

# Redis
REDIS_URL=redis://localhost:6379/0

# Application Security (optional for dev)
APP_API_KEY=your-secure-api-key-here
Security Note: Set APP_API_KEY for production deployment. Without it, the application runs in development mode with no authentication.

Launch Application

# Linux/macOS
bash launch.sh

# Windows
launch.bat

These scripts will:

  • Open the project in VS Code
  • Start the frontend at http://localhost:3000
  • Start the backend at http://localhost:8000
  • Open both interfaces in your browser

Optimization Strategies

File reference: backend/routers/portfolios.py

Strategy Description Use Case
Actual Portfolio Track performance based on your real share quantities Current holdings tracking
Equal Weight Equal dollar allocation across all holdings Baseline comparison
Mean-Variance (MVO) Maximize Sharpe ratio using historical returns Risk-adjusted optimization
Minimum Variance Minimize portfolio volatility Conservative risk management
Black-Litterman Bayesian approach combining market equilibrium with investor views Sophisticated institutional approach

Performance Metrics

  • Total Return - Percentage gain/loss over period
  • Annualized Return - Expected yearly return (252 trading days)
  • Volatility - Annualized standard deviation
  • Sharpe Ratio - Risk-adjusted return (0% risk-free rate)
  • Max Drawdown - Largest peak-to-trough decline
  • Rolling Sharpe (20d, 60d) - Time-varying risk-adjusted performance

Database Management

New Feature

Comprehensive database management interface for viewing, monitoring, and populating market data.

üíæ Database Manager Screenshot

Database Overview

File reference: backend/routers/database_stats.py

The system manages three databases:

  • Main Database (TimescaleDB): Fundamentals, daily prices, news, dividends, analyst ratings
  • Intraday Database (TimescaleDB): Minute-level OHLCV data (1m, 5m, 1h granularities)
  • Portfolios Database (SQLite): User portfolios and holdings

Database Statistics

Real-time statistics for each database:

  • Total assets (tickers) tracked
  • Total records stored
  • Date range coverage (from/to)
  • Database size on disk
  • Record counts by category (fundamentals, prices, news, etc.)

Dataset Population

File reference: backend/routers/dataset_populate.py

Populate databases with:

  • ETF Constituents: Automatically populate from SPY, QQQ, DIA, IWM, etc.
  • Index Constituents: S&P 500, NASDAQ 100, Dow Jones, Russell 2000
  • Manual Tickers: Add individual tickers or comma-separated lists
  • Asset Types: Stocks, ETFs, Forex pairs

Data Types Populated:

  • Company metadata (name, sector, industry, market cap)
  • Daily OHLCV data (historical prices)
  • Fundamentals (financial metrics) - stocks only
  • News articles with sentiment analysis
Note: Fundamentals are only available for stocks, not ETFs or Forex pairs due to EODHD API limitations.

Portfolio Analysis

GET /api/portfolios/{id}/analysis/actual
Analyze actual portfolio performance based on current share quantities
Query params: lookback_days (default: 252)
GET /api/portfolios/{id}/analysis/equal-weight
Analyze equal-weight allocation strategy
GET /api/portfolios/{id}/analysis/optimized
Optimize portfolio allocation
Query params: method (mvo | min_variance | black_litterman)

Portfolio Risk Analysis

GET /api/portfolios/{id}/monte-carlo
Run Monte Carlo simulation
Query params: num_simulations (default: 1000), time_horizon (default: 252)
GET /api/portfolios/{id}/var
Calculate Value at Risk (VaR) and Conditional VaR
Query params: confidence_level (default: 0.95)

Security & Authentication

API Key Authentication

File reference: backend/core/auth.py

All API endpoints (except health checks) require authentication when APP_API_KEY is set in environment variables.

Development Mode: If APP_API_KEY is not set, the application runs without authentication. This is suitable for local development only.

Authentication Flow

  1. Set APP_API_KEY environment variable
  2. Include X-API-Key header in all requests
  3. Backend validates key before processing request
  4. Invalid/missing keys return 401 Unauthorized
// Frontend API call example
fetch('http://localhost:8000/api/portfolios', {
  headers: {
    'X-API-Key': 'your-api-key-here',
    'Content-Type': 'application/json'
  }
})

Rate Limiting

API endpoints implement rate limiting to prevent abuse:

  • Default limit: Configured per-endpoint
  • Implementation: SlowAPI library
  • Scope: Per IP address

Input Validation

File reference: backend/core/validation.py

  • Ticker validation: Format and exchange checks
  • Request size limits: 10MB maximum payload
  • SQL injection prevention: Parameterized queries via SQLAlchemy
  • Path traversal protection: File access sanitization

CORS Configuration

CORS is configured for local development and production:

ALLOWED_ORIGINS=http://localhost:3000,http://localhost:3001
